
## Quando isso ocorre?

Esse problema acontece quando a biblioteca `better-sqlite3` ou uma biblioteca que depende dela, como `quick.db`, não está atualizada.

```bash
Error: Could not locate the bindings file. Tried:
 → /application/node_modules/better-sqlite3/build/better_sqlite3.node
 [...]
```

## Por que isso ocorre?

Esse erro ocorre porque a biblioteca `better-sqlite3` não foi atualizada para a versão mais recente. Nas versões **11.x.x e anteriores** do `better-sqlite3`, a biblioteca **não era compatível com o Node.js LTS v22**.x.x.

## Como corrigir

Para corrigir esse problema, você precisa atualizar a biblioteca `better-sqlite3` para a versão mais recente. Lembre-se de atualizar também as bibliotecas que dependem do `better-sqlite3`, como o `quick.db`.

### Verificar as últimas versões das bibliotecas:
   - [npm better-sqlite3](https://www.npmjs.com/package/better-sqlite3)
   - [npm quick.db](https://www.npmjs.com/package/quick.db)


### Atualizar o `better-sqlite3` no seu `package.json`

Depois de verificar a versão mais recente da biblioteca, você pode atualizá-la no seu `package.json`. Por exemplo, se a versão atual for 11.10.0, você pode usar o seguinte:

```json Atualizar manualmente no seu package.json
"better-sqlite3": "^11.10.0",
"quick.db": "^9.1.7" // ou qualquer outra biblioteca que dependa do better-sqlite3
```

```bash Atualizar usando o terminal
npm install better-sqlite3@latest
```

### Remover arquivos para uma instalação limpa

Após atualizar o seu `package.json`, você deve remover os arquivos e pastas: `node_modules`, `package-lock.json` e a pasta `.npm`.

### Reiniciar sua aplicação

Por fim, simplesmente reinicie sua aplicação clicando no botão azul. Se você seguiu todos os passos corretamente, sua aplicação não deverá mais encontrar esse erro.

Se você continuar enfrentando problemas, não hesite em [entrar em contato com nossa equipe de suporte](https://squarecloud.app/sac).
