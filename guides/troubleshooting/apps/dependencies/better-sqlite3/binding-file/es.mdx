
## ¿Cuándo ocurre?

Este problema ocurre cuando la biblioteca `better-sqlite3` o una biblioteca que depende de ella, como `quick.db`, no está actualizada.

```bash
Error: Could not locate the bindings file. Tried:
 → /application/node_modules/better-sqlite3/build/better_sqlite3.node
 [...]
```

## ¿Por qué ocurre?

Este error ocurre porque la biblioteca `better-sqlite3` no se ha actualizado a la última versión. En `better-sqlite3` **11.x.x y anteriores**, la biblioteca **no era compatible con Node.js LTS v22**.x.x.

## Cómo solucionarlo

Para solucionar este problema, necesitas actualizar la biblioteca `better-sqlite3` a la última versión. Recuerda actualizar también las bibliotecas que dependen de `better-sqlite3`, como `quick.db`.

### Verificar las últimas versiones de las bibliotecas:
   - [npm better-sqlite3](https://www.npmjs.com/package/better-sqlite3)
   - [npm quick.db](https://www.npmjs.com/package/quick.db)


### Actualizar `better-sqlite3` en tu `package.json`

Después de verificar la última versión de la biblioteca, puedes actualizarla en tu `package.json`. Por ejemplo, si la versión actual es 11.10.0, puedes usar lo siguiente:

```json Actualizar manualmente en tu package.json
"better-sqlite3": "^11.10.0",
"quick.db": "^9.1.7" // o cualquier otra biblioteca que dependa de better-sqlite3
```

```bash Actualizar usando el terminal
npm install better-sqlite3@latest
```

### Eliminar archivos para una instalación limpia

Después de actualizar tu `package.json`, debes eliminar los archivos y carpetas: `node_modules`, `package-lock.json` y la carpeta `.npm`.

### Reiniciar tu aplicación

Por último, simplemente reinicia tu aplicación haciendo clic en el botón azul. Si has seguido todos los pasos correctamente, tu aplicación ya no debería encontrar este error.

Si continúas experimentando problemas, no dudes en [ponerte en contacto con nuestro equipo de soporte](https://squarecloud.app/sac).
