## Creando un Nuevo Proyecto Django

- Antes de comenzar, asegúrate de tener Python y pip instalados en tu sistema. Si aún no los tienes, puedes descargarlos del [sitio oficial de Python](https://www.python.org/) y del [sitio oficial de Pip](https://pypi.org/).
- Luego, necesitarás crear una cuenta en Square Cloud, lo cual puedes hacer desde la [página de registro](https://squarecloud.app/signup). Puedes usar tu correo electrónico para crear la cuenta.
- Finalmente, debes tener un plan de pago activo en tu cuenta. Puedes ver nuestros planes y adquirir uno según tus necesidades [aquí](https://squarecloud.app/plans).

Con Python y pip instalados, puedes crear un nuevo proyecto Django. Primero, instala Django usando pip:

\`\`\`bash
pip install django
\`\`\`

Luego, crea un nuevo archivo Python (por ejemplo, \`app.py\`) y añade el siguiente código para crear una aplicación Django básica:

\`\`\`python app.py
from django.conf import settings
from django.http import HttpResponse
from django.urls import path
from django.core.wsgi import get_wsgi_application

# Configuraciones básicas de Django
settings.configure(
    DEBUG=False,  # Desactiva el modo depuración
    SECRET_KEY='mysecretkey',  # Define tu clave secreta
    ALLOWED_HOSTS=['*'],  # Permite todos los hosts por simplicidad
    ROOT_URLCONF=__name__,  # Usa este módulo como configuración raíz de URLs
    MIDDLEWARE_CLASSES=(),  # Usa una tupla vacía para deshabilitar middleware
)

# Vista simple
def index(request):
    return HttpResponse("Hello world!")  # Retorna una respuesta HTTP sencilla

# Configuración de URLs
urlpatterns = [
    path('', index),  # Mapea la URL raíz a la vista index
]

# Aplicación WSGI
application = get_wsgi_application()

# Servidor WSGI
if __name__ == "__main__":
    from wsgiref.simple_server import make_server
    httpd = make_server('', 80, application)  # Sirve la aplicación en el puerto 80
    print("Servidor Django ejecutándose en el puerto 80...")
    httpd.serve_forever()  # Inicia el servidor
\`\`\`

En el código anterior, creamos una ruta básica que devuelve "Hello, World!" al acceder. La aplicación está configurada para ejecutarse en el puerto 80, puerto predeterminado para tráfico HTTP.

## Gestión de Dependencias con el archivo requirements.txt

El archivo \`requirements.txt\` lista todas las bibliotecas externas necesarias para tu proyecto. Debes incluir todas las librerías que use el proyecto, excluyendo las nativas como \`random\` y \`datetime\`. Este archivo debe incluirse al enviar tu aplicación a Square Cloud.

\`\`\`txt requirements.txt
django
\`\`\`

## Configurando el campo START

En el archivo de configuración de Square Cloud, el campo **START** es opcional y solo necesario si usas un script personalizado para iniciar el sitio. En el ejemplo proporcionado, el campo **START** no es necesario.

## Comprar un Plan Antes de Subir Tu Proyecto

Antes de poder subir tu proyecto a Square Cloud, es importante entender que necesitas adquirir un plan. El tipo de plan dependerá de los recursos que requiera tu aplicación. Puedes ver nuestros planes [aquí](https://squarecloud.app/plans).

<Note>**¿Te preguntas cuánta RAM y CPU necesita tu plan para hospedar una aplicación Django?** No te preocupes, estamos aquí para ayudar. Django es un framework web de alto nivel para Python que fomenta el desarrollo rápido y el diseño limpio y pragmático. Está desarrollado por expertos y se encarga de gran parte de la complejidad del desarrollo web, para que puedas concentrarte en escribir tu aplicación sin reinventar la rueda. Nuestro plan **[Standard](https://squarecloud.app/pay/standard)** ofrece **4GB** de RAM y **4vCPU**, suficiente para la mayoría de las aplicaciones Django. Sin embargo, si trabajas en un proyecto más grande y necesitas mayor estabilidad, te recomendamos el plan Pro. Con recursos adicionales, mantendrás estabilidad incluso en picos de demanda. Para adquirirlo, haz clic [aquí](https://squarecloud.app/pay/pro).</Note>
