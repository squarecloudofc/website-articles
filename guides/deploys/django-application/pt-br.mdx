## Criando um Novo Projeto Django

- Antes de começar, certifique-se de que você tenha o Python e o pip instalados no seu sistema. Se ainda não os tiver, você pode baixá-los do [site oficial do Python](https://www.python.org/) e do [site oficial do Pip](https://pypi.org/).
- Em seguida, você precisará criar uma conta na Square Cloud, o que pode ser feito pela [página de cadastro](https://squarecloud.app/signup). Você pode usar seu e-mail para criar a conta.
- Por fim, você precisa ter um plano pago ativo na sua conta. Você pode ver nossos planos e adquirir um conforme suas necessidades [aqui](https://squarecloud.app/plans).

Com o Python e o pip instalados, você pode criar um novo projeto Django. Primeiro, instale o Django usando pip:

\`\`\`bash
pip install django
\`\`\`

Em seguida, crie um novo arquivo Python (por exemplo, \`app.py\`) e adicione o seguinte código para criar um aplicativo Django básico:

\`\`\`python app.py
from django.conf import settings
from django.http import HttpResponse
from django.urls import path
from django.core.wsgi import get_wsgi_application

# Configurações básicas do Django
settings.configure(
    DEBUG=False,  # Desative o modo de depuração
    SECRET_KEY='mysecretkey',  # Defina sua chave secreta
    ALLOWED_HOSTS=['*'],  # Permita todos os hosts por simplicidade
    ROOT_URLCONF=__name__,  # Defina a configuração raiz de URLs para este módulo
    MIDDLEWARE_CLASSES=(),  # Use uma tupla vazia para desabilitar middleware
)

# View simples
def index(request):
    return HttpResponse("Hello world!")  # Retorna uma resposta HTTP simples

# Configuração de URLs
urlpatterns = [
    path('', index),  # Mapeia a URL raiz para a view index
]

# Aplicação WSGI
application = get_wsgi_application()

# Servidor WSGI
if __name__ == "__main__":
    from wsgiref.simple_server import make_server
    httpd = make_server('', 80, application)  # Serve a aplicação na porta 80
    print("Servidor Django rodando na porta 80...")
    httpd.serve_forever()  # Inicia o servidor
\`\`\`

No código acima, criamos uma rota básica que retorna "Hello, World!" quando acessada. A aplicação está configurada para rodar na porta 80, que é a porta padrão para tráfego HTTP.

## Gerenciando Dependências com o arquivo requirements.txt

O arquivo \`requirements.txt\` lista todas as bibliotecas externas necessárias para o seu projeto. Você deve incluir todas as bibliotecas que o projeto usa, excluindo bibliotecas nativas como \`random\` e \`datetime\`. Este arquivo deve ser incluído ao enviar sua aplicação para a Square Cloud.

\`\`\`txt requirements.txt
django
\`\`\`

## Configurando o campo START

No arquivo de configuração da Square Cloud, o campo **START** é opcional e necessário somente se você estiver usando um script personalizado para iniciar o site. No exemplo fornecido, o campo **START** não é necessário.

## Comprando um Plano Antes de Enviar Seu Projeto

Antes de enviar seu projeto para a Square Cloud, é importante entender que você precisa adquirir um plano. O tipo de plano dependerá dos recursos necessários para sua aplicação. Você pode ver nossos planos [aqui](https://squarecloud.app/plans).

<Note>**Está se perguntando quanta RAM e CPU seu plano precisa para hospedar uma aplicação Django?** Não se preocupe, estamos aqui para ajudar. Django é um framework Python de alto nível que incentiva desenvolvimento rápido e design limpo e pragmático. É mantido por desenvolvedores experientes e cuida de grande parte da complexidade do desenvolvimento web, para que você possa focar em escrever seu app sem reinventar a roda. Nosso plano **[Standard](https://squarecloud.app/pay/standard)** oferece **4GB** de RAM e **4vCPU**, suficiente para a maioria das aplicações Django. Porém, se você estiver trabalhando em um projeto maior e precisar de mais estabilidade, recomendamos nosso plano Pro. Com recursos adicionais, você mantém estabilidade mesmo em momentos de pico. Para adquirir, clique [aqui](https://squarecloud.app/pay/pro).</Note>

Depois de preparar seus arquivos, você pode enviá-los para a Square Cloud e hospedar seu projeto.

## Solução de Problemas

<AccordionGroup>
  <Accordion title="Domínio Personalizado" icon="globe" iconType="solid">
    Se você quiser usar um domínio próprio, observe que por padrão a URL será algo como \`https://mysite.squareweb.app/\`. O subdomínio é definido pelo campo SUBDOMAIN. Para usar um domínio como \`mysite.com\`, é necessário o plano Standard ou superior. Veja como em nosso artigo: [Como Configurar Seu Domínio](https://docs.squarecloud.app/tutorials/how-to-set-up-your-domain).
  </Accordion>

  <Accordion title="Requisitos Mínimos de RAM" icon="memory" iconType="solid">
    O mínimo de RAM para hospedar um site ou API é 512MB. Porém, dependendo do tamanho e complexidade da sua aplicação Django, pode ser aconselhável usar mais RAM para evitar o erro [LACK_OF_RAM](https://docs.squarecloud.app/faqs/troubleshooting/LACK_OF_RAM).
  </Accordion>

  <Accordion title="Acesso Temporariamente Negado" icon="ban" iconType="solid">
    Se você receber "Your access has been temporarily denied." ao tentar acessar, aguarde um tempo. Se persistir, contate-nos em https://squarecloud.app/dashboard/support.

  </Accordion>
